# 🚂 راهنمای دیپلوی روی Railway

این راهنما شما را قدم‌به‌قدم برای اجرای ربات روی Railway (۲۴ ساعته آنلاین) راهنمایی می‌کند.

---

## چرا Railway؟

- ✅ همیشه آنلاین — حتی وقتی کامپیوتر خاموش است
- ✅ دیپلوی خودکار از گیت‌هاب
- ✅ دیتابیس پایدار (Volume)
- ✅ لایه رایگان برای شروع

---

## قدم ۱ — کد را روی گیت‌هاب بگذارید

```bash
cd airdrop_monitor
git init
git add -A
git commit -m "🪂 Airdrop Monitor Bot"
```

سپس روی گیت‌هاب یه ریپازیتوری جدید بسازید و پوش کنید:

```bash
git remote add origin https://github.com/USERNAME/airdrop-monitor.git
git branch -M main
git push -u origin main
```

> ⚠️ فایل `.env` به‌خاطر `.gitignore` آپلود نمی‌شود — این درست است!

---

## قدم ۲ — ثبت‌نام در Railway

1. به [railway.com](https://railway.com) بروید
2. با گیت‌هاب وارد شوید (Login with GitHub)

---

## قدم ۳ — ساخت پروژه

1. روی **New Project** کلیک کنید
2. **Deploy from GitHub repo** را انتخاب کنید
3. ریپازیتوری `airdrop-monitor` را انتخاب کنید

---

## قدم ۴ — تنظیم متغیرها (Variables)

در تب **Variables** این‌ها را اضافه کنید:

| متغیر | مقدار |
|-------|-------|
| `BOT_TOKEN` | توکن ربات از BotFather |
| `DATABASE_PATH` | `/data/data.db` |

> بقیه متغیرها اختیاری هستند و مقدار پیش‌فرض دارند.

---

## قدم ۵ — اضافه‌کردن Volume (برای دیتابیس پایدار)

1. در تب **Settings** → **Volumes** → **Add Volume**
2. مسیر Mount: `/data`
3. این باعث می‌شود دیتابیس موقع ری‌دیپلوی از بین نرود

---

## قدم ۶ — دیپلوی!

1. روی **Deploy** کلیک کنید
2. چند ثانیه صبر کنید — ربات ساخته و اجرا می‌شود
3. در تب **Deployments** → **Logs** می‌توانید لاگ‌ها را ببینید

اگر این پیام را دیدید، یعنی موفق بوده:
```
✅ ربات در حال اجراست!
```

---

## ✅ تمام!

ربات شما حالا ۲۴ ساعته آنلاین است. 🎉

برای تست، در تلگرام به ربات `/start` بفرستید.

---

## 🔧 مدیریت

| کار | روش |
|----|-----|
| دیدن لاگ‌ها | Railway → Deployments → Logs |
| ری‌استارت | Railway → Deployments → Redeploy |
| تغییر کد | روی گیت‌هاب پوش کنید — Railway خودکار آپدیت می‌شود |
| تغییر توکن | Railway → Variables → ویرایش |

---

## ❓ عیب‌یابی

| مشکل | راه‌حل |
|------|--------|
| ربات اجرا نمی‌شود | لاگ‌ها را چک کنید — احتمالاً BOT_TOKEN غلط است |
| دیتابیس ریست می‌شود | Volume را اضافه کنید (قدم ۵) |
| ربات قطع می‌شود | Railway رایگان هفته‌ای ۵۰۰ ساعت دارد — کافی است |
| `Conflict` ارور | ربات قبلی روی کامپیوتر را ببندید |
